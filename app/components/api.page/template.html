It's so easy to use the EarByter API!

BASIC SEARCH

EarByter's API will return an array of urls. These Urls point to sounds in a sound database. A typical API call would look like:

https://earbyter.herokuapp.com/api/sounds

and a developer can expect a result of the latest 100 sounds in JSON looking like:

SEARCH CRITERIA

There are 3 criteria from which a developer may search: name, genre and author. With a name search the first items matched are exact matches, the second are items that have a name "like" the input criteria. The third level of matches are "like" matches from the genre field.

A search for a particular name would look like:

https://earbyter.herokuapp.com/api/sounds?name=YOUR_NAME_HERE


A search for the genre is much like a search for name except that the genre is given first priority. A search for a particular genre would look like:

www.earbyter.com/api/sounds?genre=YOUR_GENRE_HERE


An author search will return exact matches first and "like" matches second. A search for a particular author would look like:

https://earbyter.herokuapp.com/api/sounds?author=AUTHOR_HERE

HOW TO USE THIS IN YOUR APP

A basic way to get a sound to play would be adding the URL into a source tag for an audio file like so:


{{$ctrl.htmlStr1}}

{{$ctrl.htmlStr2}}

{{$ctrl.htmlStr3}}


...where the aoundArray is the array of objects returned from the API. Each sound will have either an 'id' property or a 'sound_id' property which can be targeted for uniqueness.

Once an element has a unique id it may be targeted with an event listener to play like so:


var audio = document.getElementById(some_id);

audio.play();
